# Topic映射表

## 1. /params Topic映射 (Float32MultiArray)

**说明：** 支持两种格式，推荐使用带掩码的新格式。

### 新格式（推荐）：带参数掩码
**消息格式：** [mask_0, mask_1, mask_2, mask_3, mask_4, param_0, param_1, param_2, param_3, param_4]
- 前5位：参数更新掩码（0=不更新，1=更新）
- 后5位：参数值（0-1范围）

| 索引 | 参数名称 | 映射关系 | 输入范围 | 实际范围 | 默认值 | 描述 |
|------|----------|----------|----------|----------|--------|------|
| 0-4 | 掩码 | 更新标志 | 0或1 | - | - | 参数更新掩码 |
| 5 | leg_length | 0.14 + data[5]*(0.36-0.14) | 0-1 | 0.14 ~ 0.36 | 0.0 | 腿部长度 |
| 6 | pitch_angle | -60° + data[6]*(-60°) | 0-1 | -120° ~ -60° | 0.5 | 俯仰角度 |
| 7 | smallest_period | 0.1 + data[7]*0.3 | 0-1 | 0.1 ~ 0.4 | 0.5 | 步态最小周期 |
| 8 | stride | 0.05 + data[8]*0.15 | 0-1 | 0.05 ~ 0.2 | 0.5 | 步长 |
| 9 | z_swing | 0.02 + data[9]*0.06 | 0-1 | 0.02 ~ 0.08 | 0.5 | 摆动高度 |

### 使用示例

**示例1：只更新腿部长度**
```
消息内容：[1, 0, 0, 0, 0, 0.5, 0, 0, 0, 0]
含义：只更新leg_length为0.5，其他参数保持不变
```

**示例2：同时更新多个参数**
```
消息内容：[1, 1, 0, 1, 0, 0.3, 0.7, 0, 0.8, 0]
含义：更新leg_length=0.3, pitch_angle=0.7, stride=0.8，其他参数保持不变
```

**示例3：更新所有参数**
```
消息内容：[1, 1, 1, 1, 1, 0.0, 0.5, 0.5, 0.5, 0.5]
含义：更新所有参数为默认值
```

### 旧格式（兼容）：直接参数
**消息格式：** [param_0, param_1, param_2, param_3, param_4]

| 索引 | 参数名称 | 映射关系 | 输入范围 | 实际范围 | 默认值 | 描述 |
|------|----------|----------|----------|----------|--------|------|
| 0 | leg_length | 0.14 + data[0]*(0.36-0.14) | 0-1 | 0.14 ~ 0.36 | 0.0 | 腿部长度 |
| 1 | pitch_angle | -60° + data[1]*(-60°) | 0-1 | -120° ~ -60° | 0.5 | 俯仰角度 |
| 2 | smallest_period | 0.1 + data[2]*0.3 | 0-1 | 0.1 ~ 0.4 | 0.5 | 步态最小周期 |
| 3 | stride | 0.05 + data[3]*0.15 | 0-1 | 0.05 ~ 0.2 | 0.5 | 步长 |
| 4 | z_swing | 0.02 + data[4]*0.06 | 0-1 | 0.02 ~ 0.08 | 0.5 | 摆动高度 |

## 2. /state Topic映射 (Int8)

**说明：** 通过HTML界面按钮触发，发送对应的整数值到topic。默认值为0（或不发送）。

| 触发值 | 目标状态 | 描述 | HTML按钮 | 默认值 |
|--------|----------|------|----------|--------|
| 1 | FLIP | 触发空翻动作 | 空翻按钮 | 0 |
| 2 | JUMP | 触发前跳动作 | 前跳按钮 | 0 |
| 3 | RECOVERY | 触发恢复动作 | 恢复按钮 | 0 |
